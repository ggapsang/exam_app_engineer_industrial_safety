# 산업안전기사 기출 앱 — 디자인 가이드 (시안 A · 현장)

**버전** v1.0 · **대상** Android (API 26+) · **방향** 기관형 카드 UI (Navy + Safety Amber)
**용도** 외주 개발 인수용 디자인 스펙. 색·타이포·컴포넌트·상태·접근성을 토큰 단위로 고정한다.

> 본 가이드는 `android_app_plan.md`의 기능 요구사항(F1~F17)과 비기능 요구사항(세로 고정, 색맹 친화, 폰트 3단계, 다크모드)을 전제로 한다. 화면 전환 200ms·문항 로드 100ms 기준은 디자인이 아닌 구현 책임이며, 본 문서는 그 기준을 깨지 않는 범위의 시각 스펙만 정의한다.

---

## 1. 디자인 원칙

1. **신뢰감 우선** — 자격증 학습 앱은 화려함보다 정확성·가독성이 먼저다. 강조는 앰버 한 색에만 집중하고 나머지는 차분하게 둔다.
2. **한 화면 한 동작** — 빠른 시작(이어풀기/오답노트/모의고사)을 홈 최상단에 두어 자투리 시간 진입 비용을 최소화한다.
3. **정오는 색과 기호를 동시에** — 색맹 친화 요건상 정답/오답을 색만으로 구분하지 않는다(색 + 번호 + 아이콘).
4. **오프라인 일관성** — 네트워크 상태에 따른 UI 분기 없음. 로딩 스피너·스켈레톤 최소화(로컬 DB라 즉시 표시 가능).
5. **세로 고정** — 이미지(FT도·연쇄도) 가독성을 위해 portrait 전용. 가로 레이아웃 미정의.

> **다크 전환 원칙(필수)**: 컨테이너 배경만 어둡게 바꾸지 말고, 그 위 전경(텍스트·아이콘) 색도 반드시 함께 매핑한다. 라이트 전용으로 정의된 진한 텍스트색(`#5A4A28`, `ink` 등)을 다크 배경에 그대로 두면 어두운 글씨 + 어두운 배경으로 보이지 않는다. 특히 **밝은 배경값(`#FFF6E6` 등)을 다크에서 그대로 쓰지 말 것** — 이 경우 다크의 밝은 `ink`(`#E7EEF5`)와 충돌해 글씨가 사라진다.

### 2.3 선택지 상태 — 다크 전용 값

다크에서는 **선택지 배경을 어둡게 + 텍스트는 항상 밝은 `ink`(`#E7EEF5`)** 로 고정하고, 상태는 테두리 색·채운 번호·기호(✓/✕)로만 구분한다(색맹 친화 3중 표시 유지).

| 상태 | 배경(다크) | 테두리(다크) | 번호 원 | 텍스트 | 기호 |
| :-- | :-- | :-- | :-- | :-- | :-- |
| 기본 | `card #1A222D` | `line #2A3441` | 테두리 `muted` / 글자 `muted` | `#E7EEF5` | — |
| 선택(채점 전) | `#2E2A1A` | `amber #F2B04A` | 채움 `amber` / 글자 `#0F141B` | `#E7EEF5` | "선택" 라벨 |
| 정답 | `#13312A` | `green #2DBE8C` | 채움 `green` / 글자 `#07140F` | `#E7EEF5` | `✓` `#2DBE8C` |
| 오답 | `#3A1F1B` | `red #F0735E` | 채움 `red` / 글자 `#1A0E0B` | `#E7EEF5` | `✕` `#F0735E` |

> 채운 번호 원의 **글자색은 어두운 톤**(밝은 채움 위 가독)으로 둔다. 흰 글자는 밝은 앰버/그린 위에서 대비가 부족할 수 있다.

### 3.1 타입 스케일 (폰트 "보통" 기준, sp)

> v1.0의 기준값이 전반적으로 작아 "크게"가 이전 "작게" 수준이었다. 아래는 본문을 키운 교정값(v1.1)이며, `safety_app_mockups_v2.html`이 이 값을 반영한다.

| 역할 | 크기 | 굵기 | 행간 | 사용처 |
| :-- | :-- | :-- | :-- | :-- |
| Title | 20 | 800 | — | 앱바 제목 |
| Score | 54 | 800 | 1.0 | 모의고사 총점 |
| Stat | 22 | 800 | — | 통계 수치 |
| Stem | 17 | 600 | 1.55 | 문항 본문 |
| Choice | 16 | 500 | 1.45 | 보기 |
| Body/해설 | 15 | 400 | 1.55 | 해설 본문 |

> 최소 가독선: 본문(Stem/Choice/해설)은 "작게" 단계에서도 14sp 이상을 유지한다(위 기준 × 0.9 적용 시 충족).

### 3.2 폰트 3단계 (F10)

전체 텍스트에 **배율(scale factor)**을 곱한다. OS 글꼴 크기 설정과 별개의 앱 내부 설정.

| 단계 | 배율 | Stem 환산 | Choice 환산 |
| :-- | :-- | :-- | :-- |
| 작게 | 0.9 | 15.3sp | 14.4sp |
| 보통 | 1.0 | 17sp | 16sp |
| 크게 | 1.25 | 21.3sp | 20sp |

> 단계 폭을 넓혀 "크게"가 확실히 커지도록 했다(v1.0의 1.15 → 1.25). 기준값 자체도 키웠으므로 "작게"도 본문 14sp 이상으로 충분히 읽힌다.

> 고정 높이 컴포넌트(앱바·상태바·하단탭·아이콘 버튼)는 배율을 적용하지 않는다. 본문·선택지·해설·라벨 등 텍스트만 스케일링하며, 컨테이너는 내용에 따라 늘어나도록(wrap) 둔다. 번호 원처럼 텍스트를 감싸는 요소는 글자와 함께 비례 확대하되 최소 크기(20dp)를 보장한다.

## 5. 컴포넌트 스펙

### 5.1 앱바 (App Bar)

- 높이 48dp + 상태바, 배경 `navy`, 텍스트 흰색.
- 좌: 뒤로가기 아이콘(상세 화면) / 없음(홈은 제목만).
- 중앙 또는 좌: 제목 16sp/800, 하단에 메타 10sp `#9FB0D0`.
- 우: 아이콘 버튼(설정·즐겨찾기·닫기). 아이콘 버튼 배경 `rgba(255,255,255,.12)`, 30×30, 반경 9.

### 5.2 빠른 시작 카드 (홈 상단)

- 배경: `navy → navy-2` 135° 그라데이션, 반경 15, 패딩 14.
- 라벨 "빠른 시작" 10sp/800 `#C7D2E6`.
- 3분할 버튼: 오답노트 N / 이어풀기 % / **모의고사**(앰버 강조).
  - 일반 버튼 배경 `rgba(255,255,255,.1)`, 수치 16sp/800 `amber`.
  - 강조 버튼 배경 `amber`, 수치·라벨 `navy`.

### 5.3 모드 카드 (2열 그리드)

- 셀: 흰 배경, `1px line`, 반경 14, 패딩 11.
- 구성: 아이콘 박스(30×30, 배경 `chip`, 반경 9) → 제목 11.5sp/700 → 메타 8.5sp `muted`.
- 모드별 아이콘 색은 `navy`, 단 오답노트만 `red` 계열 강조.

### 5.4 칩 (과목/필터)

- pill(반경 20), 배경 `chip`, 텍스트 `navy-2` 9.5~10sp/700, 패딩 4×9.

### 5.5 선택지 (Choice) — 상태 정의

| 상태 | 테두리 | 배경 | 번호 원 |
| :-- | :-- | :-- | :-- |
| 기본 | `line` 1.5 | white | 테두리 `muted`, 글자 `muted` |
| 선택(채점 전) | `amber` 1.5 | `#FFF6E6` | 채움 `amber` / 글자 `navy` |
| 정답 | `green` 1.5 | `#EAFAF3` | 채움 `green` / 글자 white |
| 오답(내가 고른 틀린 답) | `red` 1.5 | `#FDEEEB` | 채움 `red` / 글자 white |

- 번호 원: 17~18dp, 안에 1~4. **색맹 대비를 위해 정답은 ✓, 오답은 ✕ 아이콘을 번호 옆 또는 우측에 추가**(색 + 번호 + 기호 3중 표시).
- 선택지 행 반경 11, 내부 패딩 9, 보기 간 간격 8.
- **위 표는 라이트 기준이다. 다크모드 값은 반드시 §2.3을 따른다.** 다크에서는 선택지 배경을 어둡게 두고 텍스트는 항상 밝은 `ink`로 고정하며, 위 라이트의 밝은 배경값(`#FFF6E6`·`#EAFAF3`·`#FDEEEB`)을 그대로 쓰지 않는다.

### 5.6 해설 카드 (F6)

- 배경 `#FFF7E9`, 테두리 `#F6E2B8`, 반경 13, 패딩 12.
- 헤더 "정답 ③ 해설" 11sp/800 `amber-d` + 정보 아이콘.
- 본문 11sp/1.55 `#5A4A28`. Markdown 일부 지원: **굵게**, 줄바꿈, 리스트.
- 참고줄(법령/공식): 상단 점선 `#ECD9AA`, 9.5sp `#9A7B3E`.
- 하단: 즐겨찾기 토글 ☆, 메모 입력 영역.

### 5.7 이미지 영역 (그림 문제, F6)

- `question_images.role='stem'` → 문항 본문 **위**에 표시.
- `role='choice1~4'` → 해당 보기 행 위에 인라인 표시.
- 컨테이너: 반경 12, 최대 폭 = 카드 내폭, 비율 유지(`fit: contain`), 배경 흰색.
- GIF 정상 재생(애니메이션 GIF가 아니어도 디코딩 보장). 로드 실패 시 "이미지를 불러올 수 없습니다" 플레이스홀더(앱 크래시 금지).
- 비인수 조건: 매핑된 204장 중 미표시 발생 시 결함.

### 5.8 진행률 (Progress)

- 트랙: 높이 5dp, 배경 `chip`, 반경 4.
- 채움: `amber`. 값 = 현재문항 / 총문항.
- 앱바 하단 또는 본문 상단 고정.

### 5.9 막대 그래프 + 합격선 (모의고사 결과 · 통계)

- 트랙 높이 14dp, 배경 `chip`, 반경 7.
- 채움 색: 통과 `green` / 과락 임박·미달 `amber`~`red`.
- **합격선 마커**: 트랙 위 60% 지점에 2dp 세로 검은선(`#0E1626`). 과목별 합격선 시각화의 핵심 장치.
- 좌측 라벨 9.5sp/700, 우측 수치 10sp/800.

### 5.10 버튼

| 종류 | 배경 | 텍스트 | 용도 |
| :-- | :-- | :-- | :-- |
| Primary | `navy` | white | 주요 확정(다음·복습 등) |
| Accent | `amber` | `navy` | 문항 카드 "다음" 등 진행 강조 |
| Secondary | `chip` | `navy` | 보조(이전·다시 풀기) |

- 높이 ≥ 44dp(터치 타겟), 반경 12, 텍스트 11sp/800.

### 5.11 하단 탭 (Bottom Nav)

- 높이 52dp, 흰 배경, 상단 `1px line`.
- 3탭: **홈 / 통계 / 설정**. 활성 `navy`, 비활성 `muted`. 아이콘 18dp + 라벨 9sp/600.
- 문항 풀이·모의고사 진행 화면에서는 하단 탭 대신 이전/다음 액션 바로 대체.

### 5.12 메모 (F6)

- 흰 배경, `1px line`, 반경 11, 패딩 9, placeholder `muted` 10sp.
- 저장 시 `memos` 테이블 upsert. 빈 입력은 행 미생성.

### 5.13 모의고사 시작 시트 (F1 채점 방식 1회 안내)

모의고사 풀이로 진입하기 **직전**, 채점 방식 옵션을 안내하고 한 번 설정하게 하는 화면(전체 화면 또는 바텀시트).

- **구성**
  - 헤더: "모의고사 시작" + 닫기(취소).
  - 시험 요약 카드(`navy` 그라데이션): 120문항 / 150분 / 합격 60점 / 과목 과락 40점 — 4분할 수치, 수치 `amber`.
  - 옵션 행 "즉시 채점" + 토글 + 보조설명("문항마다 정답·해설을 바로 확인").
  - 안내 카드(토글 상태에 따라 전환):
    - **OFF(일괄 채점·기본·권장)** → `green` 톤 카드 + "권장" 배지. "모든 문항을 푼 뒤 한 번에 채점. 합격선·과락·시간관리까지 실전 연습."
    - **ON(즉시 채점·학습용)** → `amber` 톤 카드 + "학습용" 배지. "문항마다 정답·해설 즉시 표시. 실제 시험 환경과 다름."
  - 체크박스: "다음부터 이 안내 건너뛰기 (설정에서 변경)".
  - 하단: 취소(secondary) / **풀이 시작**(primary; 토글 ON일 땐 accent 앰버로 강조).
- **토글 스위치 스펙**: 트랙 42×24, 반경 14. ON=`amber` 트랙·노브 우측, OFF=`#CDD4E0` 트랙·노브 좌측. 노브 18dp 흰색 + 약한 그림자.
- **기본값**: 모의고사는 **일괄 채점(OFF)** 이 기본. 실제 시험을 모사하는 것이 모의고사의 목적이므로 즉시 채점은 명시적 opt-in.
- **상태 저장**: "건너뛰기" 체크 시 선택한 채점 방식을 설정값(`settings.mock_grading_mode`)으로 저장하고, 다음 진입부터 시트를 건너뛴다. 설정 화면에서 언제든 재변경·재활성화 가능.
- **회차별/과목별 풀이**와 구분: 이 시트는 모의고사 전용. 일반 풀이의 즉시/일괄은 설정의 별도 항목을 따른다.

### 5.14 풀이 중단 시트 (회차/과목 풀이 · 오답 저장 선택)

회차별·과목별 풀이를 **완료 전에 나갈 때**(뒤로가기/닫기) 뜨는 바텀시트. 이번 풀이의 오답을 오답노트에 저장할지 선택한다.

- **구성**
  - 그립 핸들 + 제목 "풀이를 중단할까요?" + 메타("2021년 1회 · 44/120 풀이 중").
  - 요약 2분할 칩: 푼 문항 N / **오답 M**(오답 수치 `red`).
  - 옵션 행 "오답 M개를 오답노트에 저장" + 토글. 보조설명은 토글 상태로 전환:
    - **ON(기본)**: "끄면 이번 풀이의 오답은 저장되지 않습니다."
    - **OFF**: "이번 풀이의 오답 M개가 오답노트에 추가되지 않습니다."
  - 정보 칩(`#EEF3FB`): "진행 위치는 자동 저장되어 **이어풀기**로 다시 시작할 수 있습니다." — 진행 저장과 오답 저장은 별개임을 명확히.
  - 체크박스: "다음부터 묻지 않기 (설정에서 변경)".
  - 하단: 계속 풀기(secondary) / 나가기. **나가기 버튼은 토글 상태를 반영** — ON이면 primary(`navy`), OFF이면 경고형("저장 없이 나가기", 배경 `#FDEEEB`·글자 `red`).
- **저장 동작**
  - 진행 위치(`progress`/이어풀기)는 **토글과 무관하게 항상 저장**.
  - 오답 저장 ON → 이번 세션에서 틀린 문항을 `wrong_answers`(오답노트)에 반영. OFF → 이번 세션 오답은 미반영(이미 다른 경로로 등록된 오답에는 영향 없음).
  - 풀이를 **끝까지 완료**한 경우는 이 시트를 거치지 않고 기존 규칙대로 오답을 저장(중단이 아니므로).
- **기본값 / 기억**: 기본 토글 ON(저장). "묻지 않기" 체크 시 선택을 설정값(`settings.session_exit_save = ask | always | never`)으로 저장하고 다음부터 시트 생략. 설정에서 재변경·재노출 가능.

### 5.15 모의고사 오답 저장 시트 (채점 후 묻기)

**문제 인식**: 모의고사는 채점 후 오답이 오답노트에 자동 수집되지 않았다. 채점 직후 사용자에게 묻고 저장하도록 한다.

- **트리거**: 모의고사 제출 → 채점 완료 직후, 결과 화면 위로 바텀시트가 올라온다(결과는 뒤에 dimmed).
- **구성**
  - 그립 + 제목 "틀린 문제를 오답노트에 저장할까요?" + 메타("2021년 1회 모의고사 · 채점 완료").
  - 요약 3분할 칩: **오답 N**(`red`) / **미응시 M**(`muted`) / 정답.
  - 옵션 토글 "미응시 M문항도 함께 저장"(기본 ON) — 보조설명 "풀지 않은 문항도 복습 대상에 포함". OFF면 저장 대상은 오답만.
  - 체크박스 "다음 모의고사부터 자동 저장 (설정에서 변경)".
  - 하단: **저장 안 함**(secondary) / **N문항 저장**(primary). 버튼 라벨의 숫자는 토글 상태에 따라 갱신(오답만 = N, 미응시 포함 = N+M).
- **저장 후 결과 화면 상태**: 결과 상단에 확인 배너(`green` 톤, 체크 아이콘) "오답노트에 K문항 저장됨 · 오답 N · 미응시 M". 하단 버튼은 **"오답노트에서 복습"** 으로 전환.
- **저장 안 함 선택 시**: 배너 없음, 하단 버튼은 "이번 회차 오답 보기"(세션 한정 복습)로 둔다. 영구 오답노트에는 미반영.
- **데이터**: 저장 시 틀린(및 옵션상 미응시) 문항을 `wrong_answers`(오답노트)에 출처(회차·모의고사) 태그와 함께 추가. 중복 문항은 오답 횟수만 증가.
- **기억**: "자동 저장" 체크 시 `settings.mock_wrong_save = ask | always(+미응시 여부) | never`로 저장하고 다음부터 시트 생략, 설정에서 변경.

> §5.14(회차 중단)와 §5.15(모의고사 채점 후)는 트리거가 다르다. 전자는 **미완료 이탈 시**, 후자는 **완료·채점 후**. 두 경로 모두 결국 오답노트로 모이며, 결과 화면의 "오답 복습"(세션 한정)과 영구 "오답노트"는 구분한다.

## 7. 상태 · 피드백

- **즐겨찾기 토글**: ☆(빈 별, `muted`) ↔ ★(채운 별, `amber`). 탭 시 즉시 `favorites` INSERT/DELETE + 가벼운 스케일 애니메이션.
- **채점 피드백**: 즉시 채점 모드(F1-a)는 선택 직후 정/오 색·기호 표시 후 "다음" 활성. 일괄 채점(F1-b)은 풀이 종료 후 일괄 표시.
- **모의고사 채점 방식 안내(§5.13)**: 모의고사 진입 시 시작 시트에서 즉시/일괄을 1회 선택. 기본은 일괄(실전형). "건너뛰기" 선택 시 설정값으로 기억하고 다음부터 시트 생략, 설정에서 재변경·재노출 가능.
- **회차/과목 풀이 중단 시 오답 저장(§5.14)**: 미완료 상태로 나갈 때 시트로 오답 저장 여부 선택(기본 저장). 진행 위치는 항상 자동 저장. "묻지 않기" 시 `settings.session_exit_save`(ask/always/never)로 기억. 완료 시에는 시트 없이 기존 규칙대로 저장.
- **모의고사 채점 후 오답 저장(§5.15)**: 채점 직후 시트로 틀린(옵션상 미응시 포함) 문항을 오답노트에 저장할지 묻는다(기본 저장 권유). 저장 시 결과에 확인 배너 + "오답노트에서 복습". 미저장 시 세션 한정 "이번 회차 오답 보기"만 제공. `settings.mock_wrong_save`로 자동화 가능.
- **타이머 경고**: 모의고사 잔여 10분 이하 시 타이머 텍스트 `red`로 전환.
- **빈 상태**: 오답노트 0개 → "아직 틀린 문항이 없습니다. 회차별 풀이로 시작해 보세요." + 회차별 풀이 진입 버튼. 빈 화면은 안내가 아니라 다음 행동 유도.

## 8. 접근성

- **색맹 친화**(필수): 정오를 색만으로 구분하지 않음 → 색 + 번호 + ✓/✕ 기호 3중.
- **터치 타겟**: 모든 탭 가능한 요소 ≥ 44×44dp.
- **폰트 연동**: 앱 내 3단계 + OS 글꼴 크기 설정 반영(텍스트 sp 단위 사용).
- **명도 대비**: 본문 텍스트 대비 ≥ 4.5:1. 앰버 위 텍스트는 `navy`/`amber-d` 사용(흰 글씨 금지).
- **포커스/스크린리더**: 선택지·버튼에 contentDescription. 정답 표시는 "정답" 텍스트 라벨 포함.

## 9. 모션

- 과하지 않게. 화면 전환은 표준 fade/slide(≤ 200ms).
- 즐겨찾기 토글: 1.0 → 1.2 → 1.0 스케일(120ms).
- 채점 표시: 색 전환 fade(100ms).
- **시스템 "동작 줄이기(reduce motion)"** 설정 시 모든 전환 애니메이션 제거.

## 11. Android 구현 매핑 (참고 · Compose 권장)

val Shapes = Shapes(
    small      = RoundedCornerShape(11.dp),   // 선택지

// pill = RoundedCornerShape(percent = 50) — 칩/배지

// Type.kt (보통 배율 기준)

## 12. 비인수 / 결함 기준 (디자인 관점)

- 그림 문제에서 이미지 미표시 → 결함.
- 정오가 색만으로 구분되어 색맹 사용자가 식별 불가 → 결함(색+기호 누락).
- 앰버 위 흰 텍스트 등 명도 대비 4.5:1 미만 → 결함.
- 폰트 "크게"에서 텍스트가 컨테이너에 잘리거나 겹침 → 결함.
- 다크모드에서 앱바/카드/선택지 토큰 미적용으로 흰 배경 잔존 → 결함.
- 다크모드에서 선택지·해설 텍스트가 라이트 전경색을 유지해 어두운 배경 위에서 식별 불가(대비 4.5:1 미만) → 결함. 특히 선택(채점 전) 상태.
