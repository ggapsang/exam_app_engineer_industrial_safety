# 산업안전기사 기출 앱 — 디자인 가이드 (시안 A · 현장)

**버전** v1.0 · **대상** Android (API 26+) · **방향** 기관형 카드 UI (Navy + Safety Amber)
**용도** 외주 개발 인수용 디자인 스펙. 색·타이포·컴포넌트·상태·접근성을 토큰 단위로 고정한다.

> 본 가이드는 `android_app_plan.md`의 기능 요구사항(F1~F17)과 비기능 요구사항(세로 고정, 색맹 친화, 폰트 3단계, 다크모드)을 전제로 한다. 화면 전환 200ms·문항 로드 100ms 기준은 디자인이 아닌 구현 책임이며, 본 문서는 그 기준을 깨지 않는 범위의 시각 스펙만 정의한다.

---

## 1. 디자인 원칙

1. **신뢰감 우선** — 자격증 학습 앱은 화려함보다 정확성·가독성이 먼저다. 강조는 앰버 한 색에만 집중하고 나머지는 차분하게 둔다.
2. **한 화면 한 동작** — 빠른 시작(이어풀기/오답노트/모의고사)을 홈 최상단에 두어 자투리 시간 진입 비용을 최소화한다.
3. **정오는 색과 기호를 동시에** — 색맹 친화 요건상 정답/오답을 색만으로 구분하지 않는다(색 + 번호 + 아이콘).
4. **오프라인 일관성** — 네트워크 상태에 따른 UI 분기 없음. 로딩 스피너·스켈레톤 최소화(로컬 DB라 즉시 표시 가능).
5. **세로 고정** — 이미지(FT도·연쇄도) 가독성을 위해 portrait 전용. 가로 레이아웃 미정의.

전체 텍스트에 **배율(scale factor)**을 곱한다. OS 글꼴 크기 설정과 별개의 앱 내부 설정.

| 단계 | 배율 |
| :-- | :-- |
| 작게 | 0.9 |
| 보통 | 1.0 |
| 크게 | 1.15 |

> 고정 높이 컴포넌트(앱바 48dp, 하단탭 52dp)는 배율을 적용하지 않는다. 본문·선택지·해설만 스케일링하며, 컨테이너는 내용에 따라 늘어나도록(wrap) 둔다.

## 5. 컴포넌트 스펙

### 5.1 앱바 (App Bar)

- 높이 48dp + 상태바, 배경 `navy`, 텍스트 흰색.
- 좌: 뒤로가기 아이콘(상세 화면) / 없음(홈은 제목만).
- 중앙 또는 좌: 제목 16sp/800, 하단에 메타 10sp `#9FB0D0`.
- 우: 아이콘 버튼(설정·즐겨찾기·닫기). 아이콘 버튼 배경 `rgba(255,255,255,.12)`, 30×30, 반경 9.

### 5.2 빠른 시작 카드 (홈 상단)

- 배경: `navy → navy-2` 135° 그라데이션, 반경 15, 패딩 14.
- 라벨 "빠른 시작" 10sp/800 `#C7D2E6`.
- 3분할 버튼: 오답노트 N / 이어풀기 % / **모의고사**(앰버 강조).
  - 일반 버튼 배경 `rgba(255,255,255,.1)`, 수치 16sp/800 `amber`.
  - 강조 버튼 배경 `amber`, 수치·라벨 `navy`.

### 5.3 모드 카드 (2열 그리드)

- 셀: 흰 배경, `1px line`, 반경 14, 패딩 11.
- 구성: 아이콘 박스(30×30, 배경 `chip`, 반경 9) → 제목 11.5sp/700 → 메타 8.5sp `muted`.
- 모드별 아이콘 색은 `navy`, 단 오답노트만 `red` 계열 강조.

### 5.4 칩 (과목/필터)

- pill(반경 20), 배경 `chip`, 텍스트 `navy-2` 9.5~10sp/700, 패딩 4×9.

### 5.5 선택지 (Choice) — 상태 정의

| 상태 | 테두리 | 배경 | 번호 원 |
| :-- | :-- | :-- | :-- |
| 기본 | `line` 1.5 | white | 테두리 `muted`, 글자 `muted` |
| 선택(채점 전) | `amber` 1.5 | `#FFF6E6` | 채움 `amber` / 글자 `navy` |
| 정답 | `green` 1.5 | `#EAFAF3` | 채움 `green` / 글자 white |
| 오답(내가 고른 틀린 답) | `red` 1.5 | `#FDEEEB` | 채움 `red` / 글자 white |

- 번호 원: 17~18dp, 안에 1~4. **색맹 대비를 위해 정답은 ✓, 오답은 ✕ 아이콘을 번호 옆 또는 우측에 추가**(색 + 번호 + 기호 3중 표시).
- 선택지 행 반경 11, 내부 패딩 9, 보기 간 간격 8.

### 5.6 해설 카드 (F6)

- 배경 `#FFF7E9`, 테두리 `#F6E2B8`, 반경 13, 패딩 12.
- 헤더 "정답 ③ 해설" 11sp/800 `amber-d` + 정보 아이콘.
- 본문 11sp/1.55 `#5A4A28`. Markdown 일부 지원: **굵게**, 줄바꿈, 리스트.
- 참고줄(법령/공식): 상단 점선 `#ECD9AA`, 9.5sp `#9A7B3E`.
- 하단: 즐겨찾기 토글 ☆, 메모 입력 영역.

### 5.7 이미지 영역 (그림 문제, F6)

- `question_images.role='stem'` → 문항 본문 **위**에 표시.
- `role='choice1~4'` → 해당 보기 행 위에 인라인 표시.
- 컨테이너: 반경 12, 최대 폭 = 카드 내폭, 비율 유지(`fit: contain`), 배경 흰색.
- GIF 정상 재생(애니메이션 GIF가 아니어도 디코딩 보장). 로드 실패 시 "이미지를 불러올 수 없습니다" 플레이스홀더(앱 크래시 금지).
- 비인수 조건: 매핑된 204장 중 미표시 발생 시 결함.

### 5.8 진행률 (Progress)

- 트랙: 높이 5dp, 배경 `chip`, 반경 4.
- 채움: `amber`. 값 = 현재문항 / 총문항.
- 앱바 하단 또는 본문 상단 고정.

### 5.9 막대 그래프 + 합격선 (모의고사 결과 · 통계)

- 트랙 높이 14dp, 배경 `chip`, 반경 7.
- 채움 색: 통과 `green` / 과락 임박·미달 `amber`~`red`.
- **합격선 마커**: 트랙 위 60% 지점에 2dp 세로 검은선(`#0E1626`). 과목별 합격선 시각화의 핵심 장치.
- 좌측 라벨 9.5sp/700, 우측 수치 10sp/800.

### 5.10 버튼

| 종류 | 배경 | 텍스트 | 용도 |
| :-- | :-- | :-- | :-- |
| Primary | `navy` | white | 주요 확정(다음·복습 등) |
| Accent | `amber` | `navy` | 문항 카드 "다음" 등 진행 강조 |
| Secondary | `chip` | `navy` | 보조(이전·다시 풀기) |

- 높이 ≥ 44dp(터치 타겟), 반경 12, 텍스트 11sp/800.

### 5.11 하단 탭 (Bottom Nav)

- 높이 52dp, 흰 배경, 상단 `1px line`.
- 3탭: **홈 / 통계 / 설정**. 활성 `navy`, 비활성 `muted`. 아이콘 18dp + 라벨 9sp/600.
- 문항 풀이·모의고사 진행 화면에서는 하단 탭 대신 이전/다음 액션 바로 대체.

### 5.12 메모 (F6)

- 흰 배경, `1px line`, 반경 11, 패딩 9, placeholder `muted` 10sp.
- 저장 시 `memos` 테이블 upsert. 빈 입력은 행 미생성.

## 7. 상태 · 피드백

- **즐겨찾기 토글**: ☆(빈 별, `muted`) ↔ ★(채운 별, `amber`). 탭 시 즉시 `favorites` INSERT/DELETE + 가벼운 스케일 애니메이션.
- **채점 피드백**: 즉시 채점 모드(F1-a)는 선택 직후 정/오 색·기호 표시 후 "다음" 활성. 일괄 채점(F1-b)은 풀이 종료 후 일괄 표시.
- **타이머 경고**: 모의고사 잔여 10분 이하 시 타이머 텍스트 `red`로 전환.
- **빈 상태**: 오답노트 0개 → "아직 틀린 문항이 없습니다. 회차별 풀이로 시작해 보세요." + 회차별 풀이 진입 버튼. 빈 화면은 안내가 아니라 다음 행동 유도.

---

## 8. 접근성

- **색맹 친화**(필수): 정오를 색만으로 구분하지 않음 → 색 + 번호 + ✓/✕ 기호 3중.
- **터치 타겟**: 모든 탭 가능한 요소 ≥ 44×44dp.
- **폰트 연동**: 앱 내 3단계 + OS 글꼴 크기 설정 반영(텍스트 sp 단위 사용).
- **명도 대비**: 본문 텍스트 대비 ≥ 4.5:1. 앰버 위 텍스트는 `navy`/`amber-d` 사용(흰 글씨 금지).
- **포커스/스크린리더**: 선택지·버튼에 contentDescription. 정답 표시는 "정답" 텍스트 라벨 포함.

---

## 9. 모션

- 과하지 않게. 화면 전환은 표준 fade/slide(≤ 200ms).
- 즐겨찾기 토글: 1.0 → 1.2 → 1.0 스케일(120ms).
- 채점 표시: 색 전환 fade(100ms).
- **시스템 "동작 줄이기(reduce motion)"** 설정 시 모든 전환 애니메이션 제거.

---

## 10. 아이콘 · 에셋

- 라인 아이콘(stroke 2dp) 통일. 18~24dp.
- 모드 아이콘: 회차별=달력/격자, 과목별=목록, 모의고사=시계, 오답노트=✕, 즐겨찾기=별, 검색=돋보기.
- 별도 일러스트 없음(용량·일관성). 빈 상태만 단색 라인 아이콘 1개 + 안내 문구.

## 11. Android 구현 매핑 (참고 · Compose 권장)

> 프레임워크는 자유지만(`android_app_plan.md` 7.1), 권장 스택(Kotlin + Compose) 기준 토큰 예시를 제공한다. 다른 스택이면 동일 값으로 환산한다.

```kotlin
// Color.kt — 라이트 토큰
val Navy      = Color(0xFF16233F)
val Navy2     = Color(0xFF1F3258)
val Amber     = Color(0xFFF5A524)
val AmberDark = Color(0xFFD98A0C)
val Green     = Color(0xFF1AA97A)
val RedWrong  = Color(0xFFE25A45)
val Ink       = Color(0xFF1A2233)
val Muted     = Color(0xFF7A8499)
val LineGray  = Color(0xFFE6E9F0)
val BgGray    = Color(0xFFF4F6FA)
val Surface   = Color(0xFFFFFFFF)
val Chip      = Color(0xFFEEF1F7)

// Shape.kt
val Shapes = Shapes(
    small      = RoundedCornerShape(11.dp),   // 선택지
    medium     = RoundedCornerShape(14.dp),   // 모드 카드
    large      = RoundedCornerShape(16.dp),   // 카드
)
// pill = RoundedCornerShape(percent = 50) — 칩/배지

// Type.kt (보통 배율 기준)
val Typography = Typography(
    titleLarge = TextStyle(fontSize = 16.sp, fontWeight = FontWeight.ExtraBold),
    bodyLarge  = TextStyle(fontSize = 12.5.sp, fontWeight = FontWeight.SemiBold, lineHeight = 19.sp), // stem
    bodyMedium = TextStyle(fontSize = 11.sp, lineHeight = 17.sp),  // 해설
    labelSmall = TextStyle(fontSize = 9.sp, fontWeight = FontWeight.SemiBold),
)
```

폰트 3단계는 `CompositionLocal`로 fontScale을 주입하거나, 토큰 sp에 배율을 곱한 별도 스케일 세트를 둔다.

## 12. 비인수 / 결함 기준 (디자인 관점)

- 그림 문제에서 이미지 미표시 → 결함.
- 정오가 색만으로 구분되어 색맹 사용자가 식별 불가 → 결함(색+기호 누락).
- 앰버 위 흰 텍스트 등 명도 대비 4.5:1 미만 → 결함.
- 폰트 "크게"에서 텍스트가 컨테이너에 잘리거나 겹침 → 결함.
- 다크모드에서 앱바/카드/선택지 토큰 미적용으로 흰 배경 잔존 → 결함.
